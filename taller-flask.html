HTTP REQUEST

El navegador envía una solicitud al servidor: ¿Me das la página /tareas?

TU CÓDIGO PYTHON

Flask recibe la solicitud, ejecuta tu función, puede consultar la BD, y prepara la respuesta.

HTTP RESPONSE

El servidor devuelve el HTML. El navegador lo muestra. Eso es todo lo que hace la web.

ANATOMIA DE UNA REQUEST
  • Metodo: GET / POST
  • Ruta: /tareas
  • Headers: info del navegador
  • Body: datos del formulario
ANATOMIA DE UNA RESPUESTA
  • Status: 200 OK, 404, 500
  • HTML: lo que ve el usuario
  • Tiempo: milisegundos
TRADUCCION RAPIDA

URL = direccion. Ruta = pagina. Funcion = respuesta. Template = HTML.

bloque 1 · framework

¿Qué es Flask?

  • Un framework web minimalista para Python
  • Vos decidís qué poner — no hay magia escondida
  • Perfecto para aprender porque cada línea se entiende
  • Usado en producción por Pinterest, LinkedIn y más
  • Un solo archivo app.py es todo lo que necesitás para arrancar
¿POR QUÉ NO DJANGO?

Django hace muchas cosas automáticamente. Para aprender, queremos entender qué pasa en cada paso. Flask te lo muestra todo.

ANALOGIA RAPIDA

Flask es como una central telefonica: cada ruta es un numero y cada funcion es quien contesta.

bloque 1 · rutas

Rutas y parámetros

Las rutas son la dirección de cada "página". Podés pasar datos dinámicos en la URL.

GET /usuario/Maria

Podés capturar partes de la URL como variables en tu función Python.

GET /usuario/<nombre>

El <nombre> en la ruta se convierte en parámetro de la función.

bloque 2 · formularios

GET vs POST

GET
  • Lee datos, no los modifica
  • Los datos van en la URL
  • /buscar?q=flask
POST
  • Envía datos al servidor
  • Datos van en el cuerpo
  • Para formularios y crear cosas
⏸ ACTIVIDAD

Crear un formulario que reciba el nombre del usuario y lo salude con POST.

templates/formulario.html
<form method="POST" action="/saludo">
  <input name="nombre" placeholder="Tu nombre">
  <button type="submit">Enviar</button>
</form>
app.py
from flask import request, render_template

@app.route('/saludo', methods=['GET', 'POST'])
def saludo():
    if request.method == 'POST':
        nombre = request.form['nombre']
        return render_template('saludo.html',
            nombre=nombre)
    return render_template('formulario.html')
TIP PRACTICO

GET es para leer, POST para cambiar. Si recargas despues de POST, Flask puede volver a enviar.

bloque 3 · ORM

SQLAlchemy ORM

Un ORM (Object Relational Mapper) te deja hablar con la base de datos usando Python puro — sin escribir SQL.

Clase Python → Tabla en la BD
Atributo titulo → Columna titulo
Objeto t = Tarea() → Fila en la tabla
⏸ HACEMOS JUNTOS
  • Agregar SQLAlchemy a app.py
  • Definir el modelo Tarea
  • Crear la tabla con db.create_all()
  • Verificar que el archivo .db se creó
app.py — modelo completo
from flask import Flask
from flask_sqlalchemy import SQLAlchemy

app = Flask(__name__)
app.config['SQLALCHEMY_DATABASE_URI'] = 'sqlite:///tareas.db'
db = SQLAlchemy(app)

# Modelo = Tabla en la BD
class Tarea(db.Model):
    id = db.Column(db.Integer, primary_key=True)
    titulo = db.Column(db.String(200), nullable=False)
    completada = db.Column(db.Boolean, default=False)

# Crear la tabla al iniciar
with app.app_context():
    db.create_all()
SQL VS ORM

ORM traduce Python a SQL. Es mas legible y evita errores repetidos.

# READ — mostrar todas las tareas
@app.route('/')
def index():
    tareas = Tarea.query.all()
    return render_template('index.html', tareas=tareas)

# CREATE — guardar nueva tarea
@app.route('/agregar', methods=['POST'])
def agregar():
    titulo = request.form['titulo']
    nueva = Tarea(titulo=titulo)
    db.session.add(nueva)
    db.session.commit()
    return redirect('/')

⏸ CRUD COMPLETO — ACTIVIDAD FINAL

Ahora tenés el ciclo completo. Probá crear, editar y eliminar tareas hasta que todo funcione sin errores.

app.py — update + delete
# UPDATE — completar tarea
@app.route('/completar/<int:id>')
def completar(id):
    tarea = Tarea.query.get_or_404(id)
    tarea.completada = not tarea.completada
    db.session.commit()
    return redirect('/')

# DELETE — eliminar tarea
@app.route('/eliminar/<int:id>')
def eliminar(id):
    tarea = Tarea.query.get_or_404(id)
    db.session.delete(tarea)
    db.session.commit()
    return redirect('/')
